This is an example of an uncommon HTML bug. It involves incorrect nesting of elements within a <details> tag.

<details>
  <summary>Details</summary>
  <p>This is some text.</p>
  <div>
    <p>This is more text.</p>
    <ul>
      <li>List item 1</li>
      <li>List item 2</li>
    </ul>
    <details> 'Nested details which is invalid '</details>
  </div>
</details>

The problem is that the second <details> element is incorrectly nested inside the first. Although the browser might not always throw an error, it can lead to unexpected behavior, especially with regards to the opening and closing of the details elements.
